## 1. Explanation of Bayes' Theorem
Bayes' Theorem is a fundamental principle in probability theory used to determine the conditional probability of an event based on prior knowledge of conditions related to that event.

The mathematical formula is expressed as:

$$P(A|B) = \frac{P(B|A) \cdot P(A)}{P(B)}$$

**Components:**
* **$P(A|B)$ :** The probability of hypothesis $A$ being true given that evidence $B$ has occurred.
* **$P(B|A)$ :** The probability of evidence $B$ being observed given that hypothesis $A$ is true.
* **$P(A)$ :** The initial probability of hypothesis $A$ before seeing the evidence.
* **$P(B)$ :** The total probability of the evidence $B$ occurring.

# 2. Bayes Theory in Naive Bayes Algorithm

Naive Bayes is a classification algorithm based on Bayes Theorem with the assumption that all features are independent.

The algorithm calculates the probability of a data point belonging to each class and chooses the class with the highest probability.

Steps of Naive Bayes:

1. Calculate prior probability for each class.
2. Calculate likelihood of each feature for each class.
3. Multiply probabilities according to Bayes theorem.
4. Select the class with the highest posterior probability.

Because of the *independence assumption*, Naive Bayes becomes computationally efficient and works well for large datasets.

## 4. Confusion Matrix

To evaluate the performance of the Naive Bayes classifier, this project uses a **Confusion Matrix**.
A confusion matrix is a performance measurement tool for classification models that compares the **actual labels** in the dataset with the **labels predicted by the model**.

Unlike accuracy alone, a confusion matrix provides a more detailed view of the model’s performance by showing **which types of predictions are correct and which are incorrect**.

In a **binary classification problem** such as spam detection, the confusion matrix is structured as follows:

| Actual / Predicted | Ham                 | Spam                |
| ------------------ | ------------------- | ------------------- |
| **Ham**            | True Negative (TN)  | False Positive (FP) |
| **Spam**           | False Negative (FN) | True Positive (TP)  |

In this table:

* **Rows represent the actual labels** from the dataset.
* **Columns represent the predicted labels** generated by the model.

Each cell in the matrix represents the number of observations that fall into that category.

---

### True Positive (TP)

A **True Positive** occurs when the model **correctly predicts a message as spam**, and the message is indeed **spam** in the dataset.

This means the model successfully detects an unwanted or malicious message.

Example:

```text
Congratulations! You have won a $1000 prize. Click the link to claim now.
```

* Actual label: **Spam**
* Predicted label: **Spam**

This type of prediction is desirable because the model successfully prevents spam messages from being treated as legitimate messages.

---

### True Negative (TN)

A **True Negative** occurs when the model **correctly predicts a message as ham**, and the message is indeed a **legitimate message**.

Example:

```text
Hey, are we still meeting at 7 pm tonight?
```

* Actual label: **Ham**
* Predicted label: **Ham**

This indicates that the model successfully identifies a normal message and does not mistakenly flag it as spam.

A high number of **True Negatives** indicates that the model is good at recognizing legitimate communication.

---

### False Positive (FP)

A **False Positive** occurs when the model **predicts a message as spam**, but the message is actually **ham**.

Example:

```text
Please send me the meeting notes when you are done.
```

* Actual label: **Ham**
* Predicted label: **Spam**

In this case, the model incorrectly flags a legitimate message as spam.

This type of mistake is called a **Type I Error**.

False positives are problematic because they may cause **important legitimate messages to be filtered or ignored**.

---

### False Negative (FN)

A **False Negative** occurs when the model **predicts a message as ham**, but the message is actually **spam**.

Example:

```text
Limited offer! Buy one get one free. Click now!
```

* Actual label: **Spam**
* Predicted label: **Ham**

This means the spam message was **not detected** by the model.

This type of mistake is known as a **Type II Error**.

False negatives are dangerous in spam detection systems because **spam messages can bypass the filter and reach users**.

## 5. Model Accuracy

After generating predictions from the Naive Bayes classifier, the model performance is measured using **accuracy**.

Accuracy represents the proportion of correct predictions among the total number of predictions made by the model.

The formula for accuracy is:

[
Accuracy = \frac{TP + TN}{TP + TN + FP + FN}
]

Where:

* **TP** = True Positives
* **TN** = True Negatives
* **FP** = False Positives
* **FN** = False Negatives

In the implementation, the accuracy is calculated using:

```python
print("Accuracy:", accuracy)
```

The model achieved an accuracy of approximately **95.61%**, meaning that **around 95% of the messages in the test dataset were correctly classified** as either spam or ham.
